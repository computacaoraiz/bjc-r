Depuração e Expansão do Seu Jogo de Adivinhação de Números

Nesta página, você praticará a depuração e fará o computador dar dicas aos jogadores.

  1. No arquivo há uma definição de um bloco jogo de adivinhação de números. Ele contém um bug—um erro no código que faz o programa se comportar de forma diferente do esperado. Este código, se você continuar jogando várias vezes, funcionará na maioria das vezes, mas às vezes (não frequentemente) ele o parabenizará antes mesmo de você adivinhar.

    Analise o código, explique como isso pode acontecer, corrija o bug e salve o projeto.

    2. Deixe-o escolher seu número aleatório apenas de 1 a 3 para que o bug ocorra com mais frequência.

    Se você quiser adicionar mais variáveis de script, use o triângulo apontando para a direita. (O triângulo esquerdo as removerá.)

  2. Agora reabra o seu projeto U2L1-NumberGuessing. Certifique-se de que sua versão não tenha o mesmo bug.

Expandindo o Programa com Respostas Mais Específicas

  1. Use mais blocos condicionais e predicados para fazer o computador dizer ao jogador se um palpite é muito grande ou muito pequeno: "Isso é muito grande. Tente novamente."
    Onde no código esses condicionais deveriam ser inseridos?
  2. Teste e depure. Jogue seu jogo o suficiente para ver se ele funciona como você deseja. Ele deve:
    • pedir ao jogador para adivinhar seu número secreto;
    • dizer se um palpite incorreto é muito grande ou muito pequeno, e depois pedir novamente;
    • parabenizar o jogador por uma resposta correta.

A variável de script número secreto existe apenas até que o script termine de ser executado. Quando o jogo é jogado novamente, o programa criará uma nova variável número secreto e lhe atribuirá um novo número aleatório.

Em uma unidade posterior, você ensinará o computador a adivinhar um número secreto que você escolheu.

  1. Neste momento, o script sempre escolhe um número entre 1 e 10. No início do jogo, pergunte ao jogador qual deve ser o número máximo, e faça o seu programa escolher um número entre 1 e máximo.
  2. Use outra variável de script para acompanhar quantas tentativas o jogador faz antes de acertar o número correto. Quando o jogador adivinhar o número secreto, diga quantas tentativas foram necessárias.
  1. Encontre uma maneira de variar um pouco a linguagem para que seu programa não repita exatamente as mesmas palavras na mesma situação sempre. Por exemplo, se um jogador der um palpite muito alto duas vezes seguidas, o programa poderia dizer "Ainda é muito grande. Tente outro número."
