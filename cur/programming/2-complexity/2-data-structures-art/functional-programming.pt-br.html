Programação Funcional

Na linguagem usada no exame, a única maneira de gerar dados de lista é com comandos, ações que alteram o conteúdo de uma lista: Adicionar< /código>. Inserir. Substituir. Excluir. Eles adotam essa abordagem porque acham que é o estilo de programação mais fácil de entender, e também por incluir o Scratch, no qual esses comandos são o único mecanismo fornecido para trabalhar com listas. Mas existe uma maneira melhor, chamada programação funcional, na qual você nunca modifica uma estrutura de dados, mas em vez disso cria uma nova para conter os novos valores.

Ainda não estou convencido de que é sempre melhor. Certamente, em alguns casos é, mas às vezes é uma grande bagunça feia e confusa. Ao mesmo tempo, a programação funcional era vista por programadores "reais" como algo de interesse apenas para professores de ciência da computação. Mas hoje é uma forma preferida de programar por causa do hardware de processamento paralelo, seja com racks cheios de computadores ou com chips de processador multicore que incluem quatro ou oito processadores dentro de um único chip. A razão é que com vários processos rodando em paralelo, é possível que dois deles tentem modificar a mesma variável ao mesmo tempo, com resultados ruins. Mas se o programa for escrito funcionalmente, nada será modificado, então isso não pode acontecer. Existem linguagens funcionais nas quais não há equivalentes para set ou replace.
