Funções Booleanas

No nível mais baixo, a circuitaria de um computador é feita de fios, e cada fio está ligado ou desligado. Portanto, as únicas operações que podem ser realizadas nesse nível mais baixo são aquelas que operam em valores de um único bit (apenas uns e zeros, ou seja, ligados e desligados). Essas são chamadas de funções lógicas (ou booleanas). (Elas são predicados, porque sua faixa é de valores booleanos, mas são especiais porque seu domínio também é booleano.) Mas ou é um pouco diferente na ciência da computação. Em inglês, a palavra "ou" tem dois significados diferentes:
  • Ou um, mas não ambos: como "Vamos verificar se estará chuvoso ou ensolarado amanhã." Na ciência da computação, isso é chamado de ou exclusivo, porque uma opção exclui a outra.
  • Ou um ou ambos: como "Pergunte ao seu professor se você tiver alguma dúvida ou se precisar de ajuda." Isso é chamado de ou inclusivo, porque é possível que ambas as opções estejam incluídas.

TTFT sim equivalente a implicação reversa (falso somente quando o primeiro input é falso mas o segundo é verdadeiro; o segundo input sendo verdadeiro implica que o primeiro input deve ser verdadeiro)
TTFF não relata o valor do primeiro input independentemente do segundo input, então você poderia usar o primeiro input como o predicado em vez de construir essa função de predicado
TFTT sim equivalente a implicação (falso somente quando o primeiro input é verdadeiro mas o segundo é falso; o primeiro input sendo verdadeiro implica que o segundo input deve ser verdadeiro)
TFTF não relata o valor do primeiro input independentemente do segundo input, então você poderia usar o primeiro input como o predicado em vez de construir essa função de predicado
TFFT sim equivalente a não-implicação reversa (falso somente quando o primeiro input é verdadeiro mas o segundo é falso; o segundo input sendo falso implica que o primeiro input deve ser falso)
TFFF sim equivalente a não (ou seja, o contrário do primeiro input)
FTTT sim equivalente a implies (falso somente quando o primeiro input é falso mas o segundo é verdadeiro; o primeiro input sendo falso implica que o segundo input deve ser verdadeiro)
FTTF sim equivalente a xor (falso somente quando ambos os inputs são iguais)
FTFT sim equivalente a não-implicação (falso somente quando o primeiro input é falso mas o segundo é verdadeiro; o segundo input sendo verdadeiro implica que o primeiro input deve ser verdadeiro)
FTFF sim equivalente a and (falso somente quando o primeiro input é falso)
FFTT sim equivalente a não-implies (falso somente quando ambos os inputs são iguais)
FFTF sim equivalente a nor (falso somente quando pelo menos um dos inputs é verdadeiro)
FFFT sim equivalente a biconditional (falso somente quando os inputs são diferentes)
FFFF não sempre relata false
