Nesta página, você aprenderá que, mesmo para problemas de tempo não razoável, ainda pode haver soluções "bastante razoáveis" em termos de tempo.

AAP-4.A.6, AAP-4.A.8, AAP-4.A.9

Se um problema pode ser resolvido em tempo exponencial, pode haver um algoritmo diferente que pode fazê-lo em tempo polinomial (ou seja, mais rapidamente), mas alguns problemas não podem ser respondidos em tempo polinomial de forma alguma. É importante reconhecer que um algoritmo de tempo exponencial ainda resolve corretamente um problema; apenas leva um tempo irracionalmente longo (talvez até centenas de anos para algumas entradas, por exemplo).

Algoritmos de tempo exponencial às vezes podem ser substituídos por heurísticas, que são algoritmos de tempo polinomial que não resolvem o problema exatamente, mas fornecem uma aproximação suficientemente boa. Mas heurísticas são úteis apenas para certos tipos de problemas.

AAP-4.A.2
  • Um problema de decisão é um problema com uma resposta verdadeira/falsa (por exemplo, "é 5.825.496.221 um número primo?").
  • Um problema de otimização é aquele com o objetivo de encontrar a melhor solução entre muitas (por exemplo, "qual é o melhor horário escolar para encaixar todos os alunos em quantas aulas eles pediram?").
  1. Para qual tipo de problemas (problemas de otimização ou problemas de decisão) uma solução heurística é provavelmente útil?

Um exemplo de um problema para o qual uma solução heurística é útil é o problema do caixeiro-viajante: Para um grupo de cidades, qual é a rota mais curta para um vendedor visitar todas as cidades e retornar à cidade de origem? Este é um bom caso para heurísticas porque:

  1. É claro que deve haver uma rota mais curta.
  2. Não há nenhum algoritmo de tempo polinomial conhecido para esse problema. (A maioria dos cientistas da computação acredita que não é possível escrever um.)
  3. Existe uma heurística possível: escolher um caminho aleatoriamente e tentar melhorá-lo trocando duas cidades repetidamente até que não seja possível fazer o caminho melhor com essas pequenas mudanças.
Essa heurística é chamada de "subir a colina" porque você encontrará o melhor caminho próximo (o topo de uma colina), mas pode haver uma colina mais alta (um caminho melhor) mais distante.

Podem Todos os Problemas de Decisão Serem Resolvidos?

Como acontece, nem todos os problemas de decisão (questões verdadeiras/falsas) podem ser resolvidos com um algoritmo.

AAP-4.B.1, AAP-4.B.2, AAP-4.B.3

Um problema decidível é um problema de decisão para o qual é possível escrever um algoritmo que fornecerá uma saída correta para todas as entradas.

Um problema indecidível é o oposto. Não é possível escrever um algoritmo que fornecerá uma saída correta para todas as entradas - mesmo que seja possível para algumas delas.

A pergunta "O número inteiro é par?" é um exemplo de um problema decidível porque é possível escrever um algoritmo que determinará se qualquer número inteiro é par.

A pergunta "Este programa de computador que recebe uma entrada sempre eventualmente reportará um resultado?" é um problema indecidível. É possível escrever um algoritmo de verificação que dirá se alguns programas com algumas entradas relatarão um resultado ou ficarão presos em um loop infinito e nunca relatarão. Mas acontece que não é possível escrever um algoritmo de verificação que funcionará para qualquer programa com qualquer entrada. (Você pode ver uma prova de que nenhum algoritmo de verificação desse tipo - para um computador ou uma pessoa - pode ser escrito no Laboratório 4.)
