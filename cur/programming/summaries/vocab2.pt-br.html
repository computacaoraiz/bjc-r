2.2.2: Tipos de dados abstratos
    AAP.1.D.1

    A palavra "abstrato" é frequentemente usada casualmente para significar algo mais difícil de entender ou mais complexo, mas na ciência da computação seu significado é quase o oposto. ADTs são coisas que você, o programador, cria para tornar seu programa mais fácil para você e outras pessoas lerem, depurarem e melhorarem.

  • Um tipo de dados abstrato (ADT) é um tipo de dados personalizado que é significativo para o seu programa. Não está embutido na linguagem; você o desenvolve à medida que codifica. Isto é uma abstração porque esconde os detalhes dentro do construtor e dos seletores para que o programador que usa um item de questionário tenha que pensar apenas em perguntas e respostas, não em índices de lista.
  • O bloco é o construtor; ele constrói um exemplo da estrutura de dados. Ao nomear as peças da estrutura, o construtor garante que cada vez que você usá-lo, você construirá a estrutura na ordem certa.
  • A e são os seletores; cada um deles seleciona uma parte da estrutura de dados.
O construtor e o seletor implementam juntos o item do questionário tipo de dados abstrato.

2.2.2: Tabela  

Uma tabela é uma tabela bidimensional estrutura de dados com linhas e colunas. Se você tiver usou um programa de planilha, o que exibe é uma tabela.

No Snap!, uma tabela é implementada como uma lista de listas, em que cada sublista é uma linha da tabela.

2.2.3: Percorrendo uma lista  
AAP-2.O.2

Percorrer uma lista significa olhar cada item da lista. For each é iterativo. Ou seja, é repetitivo, como for, que também pode percorrer uma lista. Mas, diferentemente de for, for each percorre a lista sem usar números de índice.

Laboratório 3: Tomando decisões

2.3.1: Domínio e Intervalo  
  • O tipo de entrada (geralmente chamado de domínio pelos programadores) de uma função é o tipo de dados que ela aceita como entrada.
  • O tipo de saída (geralmente chamado de intervalo) de uma função é o tipo de dados que ela reporta como saída.
2.3.1: Sequenciamento, Seleção, Iteração  
Seleção: AAP-2.G.1; sequenciamento, seleção, iteração: AAP-2.A.4

Seleção significa decidir (selecionar) qual parte de um algoritmo será executada com base no fato de uma condição ser verdadeira ou falsa.

Todo algoritmo pode ser construído usando sequenciamento (seguindo etapas em ordem), seleção (decisão) e iteração (repetição).< /p>

2.4. 2: APIs  
AAP-3.D.4, AAP-3.D.5

Uma interface de programa de aplicação (API) documenta o que um programador precisa saber sobre o uso de uma biblioteca: é uma descrição da finalidade, entradas e saídas de cada procedimento (mas não seus algoritmos).

O que é uma API Web?

Um tipo comum de API é uma API web na qual a biblioteca existe no computador de outra pessoa. Por exemplo, a API do Google Maps descreve como incorpore um mapa do Google em seu próprio site.
