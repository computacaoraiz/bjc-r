1.2.3 : Listas, Strings e Concatenação
    AAP-1.C.1 primeira frase
  • O que significa "ordenado"?
    Isso não significa que os itens precisam aparecer em ordem alfabética ou numérica. Isso apenas significa que é uma lista diferente de .
    Uma lista é uma sequência ordenada de itens. Você já viu esse exemplo:
    • AAP-2.D.1, AAP-2.D.2
  • Os itens desta lista são strings. Um string é uma sequência de caracteres (letras, dígitos, pontuação, etc.). Um substring é apenas uma parte de um string existente. Por exemplo, "Hanna," "anna" e "nnah" são cada um subconjunto do string "Hannah." (O string vazio e o string original também são ambos substrings.)
  • Concatenar strings significa criar um string maior conectando dois ou mais strings menores. Em Snap!, o bloco permite que você concatene strings.

1.2.4 : Procedimentos, Reporters e Comandos
AAP-3.A.1, AAP-3.A.2

Um procedimento é uma sequência nomeada de instruções que pode receber entradas e pode reportar um valor. Algumas linguagens chamam procedimentos de métodos ou funções. Aqui estão dois tipos de procedimentos que você viu no Snap!:

  • Reporters têm uma forma oval. Eles reportam um valor.
  • Comandos têm uma forma de quebra-cabeça de encaixe. Eles dizem ao computador para fazer algo sem reportar um valor.

AAP-2.B.3, AAP-2.B.4
  • Uma expressão é um valor constante (como "4" ou "inverno") ou uma chamada a um bloco de reporter incluindo suas entradas (como , ou ).
  • As expressões são avaliadas para produzir um único valor (um valor pode ser um número, uma string, um sprite, um traje, um script, uma lista - qualquer coisa). Por exemplo, será avaliado como 17.

Laboratório 3: Arte Moderna com Polígonos

1.3.3 : O que é uma entrada? Parâmetro vs. Argumento
AAP-3.A.3

  • Um parâmetro (ou parâmetro formal) é o nome da entrada, como número de ramos. O nome da entrada é definido na definição do bloco. Ele nunca muda.
  • Um argumento (ou argumento real) é o valor da entrada, como 6 para um cata-vento hexagonal. O valor de entrada é fornecido cada vez que o bloco é executado; ele pode ser um valor diferente a cada vez.
Usamos a palavra "entrada" tanto para parâmetros (nomes de entrada) quanto para argumentos (valores de entrada).
