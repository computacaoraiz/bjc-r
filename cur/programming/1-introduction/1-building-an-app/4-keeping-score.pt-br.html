Dar feedback ao jogador

Agora o jogo está mais difícil, mas os jogadores não saber dizer se estão indo bem porque o Alonzo fica pulando se o jogador conseguir acertar o clique nele ou não. Nesta página, você criará uma maneira para o jogador saber como está se saindo.

Confirmando se o Alonzo foi clicado

Com certeza a pessoa que está jogando o seu jogo quer saber, imediatamente, se ela conseguiu clicar no Alonzo antes que ele pulasse para outro lugar.

: Personagens (Sprites) e Fantasias (Costumes)

O personagem Alonzo tem esse nome por causa do Alonzo Church, um grande contribuidor na era inicial da ciência da computação. Neste projeto existem três objetos relacionados ao Alonzo:

  • O primeiro é o personagem (sprite), que é como se fosse um ator que está no palco e sabe como fazer muitas coisas diferentes como, por exemplo, andar pelo palco e recitar as falas da peça.
  • Os outros são as fantasias (costumes), figuras que podem ser "utilizadas" pelo personagem.

Programe o Alonzo para virar para o outro lado quando for clicado.
  1. O projeto que você carregou para trabalhar já tem duas fantasias (costumes) para o Alonzo — uma com ele virado para a direita e a outra com ele virado para a esquerda — e, assim, você pode usar o bloco (próxima fantasia) para fazer o personagem virar para o outro lado. Adicione o bloco next costume (próxima fantasia) para local no seu programa que verifica (ou "sente") que o usuário acabou de clicar no Alonzo.
  2. Teste o seu programa. Certifique-se que de que o Alonzo só vira para o outro lado quando ele for clicado pelo usuário, e não quando ele ficar pulando sem ser clicado.

Mantendo uma pontuação

O jogador também irá querer saber quanto progresso ele fez no jogo. Ele está ganhando ou está perdendo o jogo? O comando pode controlar a transparência do Alonzo.

A transparência de uma imagem diz respeito ao quanto que você consegue enxergar o que está por trás dessa imagem. Por exemplo, aqui está o personagem Alonzo com três diferentes transparências (que foram ajustadas com o bloco ghost effect (efeito fantasma).

Use a transparência para informar ao usuário se ele está ganhando o jogo: toda vez que o usuário acertar o clique, o Alonzo ficará mais invisível e, quando o ele sumir completamente, o usuário ganhou o jogo.

  1. Arraste o bloco change (ghost) effect (alterar o efeito [fantasma] por) para a área central de scripts, e experimente diferentes números (clicando no bloco repetidamente) para ver exatamente o que esse bloco faz. Em qual valor do efeito fantasma o Alonzo se torna completamente invisível? Você pode usar o comando para tornar o Alonzo totalmente visível novamente.
  2. Depois, modifique seu script para que a cada vez que o Alonzo for clicado, ele se torne um pouco mais transparente. Pense como a entrada que você usar afetará a duração do jogo. Teste o seu jogo.

Deixando o jogo mais difícil

Até agora mesmo um jogador ruim irá, eventualmente fazer o Alonzo desaparecer. O jogo ficaria muito mais interessante se o jogador fosse penalizado quando errasse um clique.

  1. Descubra onde adicionar um bloco change (ghost) effect by (-5) (alterar o efeito [fantasma] por (-5)) no script de seu programa para tornar o Alonzo menos transparente toda vez que ele se mover sem ter sido clicado.
  2. Jogue você mesmo o seu jogo algumas vezes, e ajuste o código de seu jogo até alcançar um nível de dificuldade que você goste.
    3. Você pode alterar o número informado ao bloco change effect e/ou o time no bloco wait para tornar seu jogo mais fácil ou mais difícil.
